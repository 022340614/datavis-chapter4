GitHub Pages 404问题诊断

步骤1：检查仓库名称

确认GitHub仓库名称是否为：datavis-chapter4

访问：https://github.com/022340614/datavis-chapter4

步骤2：检查GitHub Pages设置

在仓库Settings → Pages中确认：

  • Source: Deploy from a branch
  • Branch: main
  • Folder: / (root)
  • 显示绿色勾选标记

步骤3：检查文件结构

确保以下文件在仓库根目录：

  • index.html
  • styles.css
  • script.js
  • .nojekyll

步骤4：强制重新部署

在仓库Actions中手动运行"Deploy to GitHub Pages"工作流

正确访问地址

https://022340614.github.io/datavis-chapter4/
